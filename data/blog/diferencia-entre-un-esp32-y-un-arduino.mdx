## Diferencias Generales

| Aspecto                        | ESP32                           | Arduino                             |
| ------------------------------ | ------------------------------- | ----------------------------------- |
| **CPU**                        | Xtensa Dual Core LX6 de 32 bits | ATMega328P (Uno), ATMega2560 (Mega) |
| **Velocidad de Reloj**         | Hasta 240 MHz                   | 16 MHz (Uno)                        |
| **Memoria Flash**              | 4MB                             | 32KB (Uno)                          |
| **SRAM**                       | 520KB                           | 2KB (Uno)                           |
| **Voltaje de Operación**       | 3.3V DC                         | 5V DC                               |
| **Consumo de Corriente**       | 80 mA - 90 mA                   | 45 mA - 80 mA (Uno)                 |
| **Pines de E/S Digitales**     | 36                              | 14 (Uno)                            |
| **Pines de Entrada Analógica** | Hasta 18                        | 6 (Uno)                             |
| **UARTs**                      | 3                               | 1 (Uno)                             |
| **SPI**                        | 4                               | 1 (Uno)                             |
| **I2C**                        | 2                               | 1 (Uno)                             |
| **Conectividad Inalámbrica**   | Wi-Fi y Bluetooth integrados    | Requiere módulos adicionales        |
| **Aplicaciones**               | IoT, domótica, drones           | Educación, prototipos rápidos       |

### CPU y Velocidad de Reloj

El **ESP32** cuenta con un microprocesador **Xtensa Dual Core LX6 de 32 bits**, que puede operar hasta **240 MHz**¹. Esto le permite realizar múltiples tareas y manejar operaciones más complejas con mayor eficiencia. En contraste, los Arduinos como el Uno o el Mega utilizan microcontroladores **ATMega328P** y **ATMega2560** respectivamente, con velocidades de reloj de **16 MHz**¹.

### Memoria y Almacenamiento

El ESP32 supera a los Arduinos en términos de memoria y almacenamiento. Por ejemplo, el ESP32 tiene **4MB de memoria flash** y **520KB de SRAM**, mientras que el Arduino Uno tiene **32KB de memoria flash** y **2KB de SRAM**¹. Esto significa que el ESP32 puede almacenar programas más grandes y manejar más datos en tiempo real.

### Consumo de Energía y Voltaje de Operación

El ESP32 opera a **3.3V DC** y tiene un consumo de corriente de **80 mA - 90 mA**. Por otro lado, Arduino opera a **5V DC** y su consumo de corriente varía entre **45 mA - 80 mA** para el Uno y hasta **150 uA** para el Mega¹. Esto es importante para proyectos alimentados por baterías, donde el menor consumo de energía del ESP32 puede ser beneficioso.

### Pines de E/S y Capacidades de Comunicación

El ESP32 ofrece **36 pines digitales de E/S** y hasta **18 pines de entrada analógica**, mientras que el Arduino Uno tiene **14 pines digitales de E/S** y **6 pines de entrada analógica**¹. Además, el ESP32 incluye **3 UARTs**, **4 SPI**, y **2 I2C**, comparado con el Arduino Uno que tiene **1 UART**, **1 SPI**, y **1 I2C**¹. Esto le da al ESP32 una mayor flexibilidad para conectar varios dispositivos y sensores.

### Conectividad Inalámbrica

Como mencionamos antes, el ESP32 tiene **Wi-Fi** y **Bluetooth** incorporados, lo que lo hace ideal para proyectos de IoT. Arduino carece de estas características de forma nativa y requiere módulos adicionales para lograr una funcionalidad similar¹.

### Aplicaciones

Debido a su mayor potencia de procesamiento y conectividad, el ESP32 es adecuado para aplicaciones más exigentes como estaciones meteorológicas inteligentes, domótica, y drones. Arduino, siendo más sencillo y con un ecosistema amplio, es excelente para la educación, prototipos rápidos y proyectos donde la conectividad no es crítica.

## Detalles de Programación

| Aspecto                       | ESP32                                                                                 | Arduino                           |
| ----------------------------- | ------------------------------------------------------------------------------------- | --------------------------------- |
| **Entorno de Desarrollo**     | Compatible con el IDE de Arduino y otros entornos                                     | IDE de Arduino y otros entornos   |
| **Lenguaje de Programación**  | C++, soporte para Python y Lua                                                        | C/C++                             |
| **Funciones de Tiempo**       | `millis()`, `micros()`, `delay()` (no bloqueante), `delayMicroseconds()` (bloqueante) | `millis()`, `micros()`, `delay()` |
| **Yielding**                  | Necesario para gestionar la conexión Wi-Fi y la pila TCP/IP, evita reinicios          | No aplica                         |
| **Designación de Pines**      | Recomendado usar números de GPIO directamente                                         | Uso de alias o números de pines   |
| **E/S Digitales (GPIO)**      | Programación similar a Arduino                                                        | Programación estándar de Arduino  |
| **Salidas Analógicas (PWM)**  | Varias formas de generar señal PWM, más complejo                                      | Más sencillo y limitado           |
| **Entradas Analógicas (ADC)** | Funcionamiento similar, pero con diferencias técnicas                                 | Funcionamiento estándar           |
| **Interrupciones**            | Soporte en todos los pines GPIO                                                       | Soporte limitado                  |

- **Funciones de Tiempo**: En el ESP32, las funciones `millis()` y `micros()` funcionan igual que en Arduino, así como `delay()` y `delayMicroseconds()`. Sin embargo, `delay` es no bloqueante en ESP32, lo que significa que otras tareas pueden ejecutarse mientras se espera, mientras que `delayMicroseconds` es bloqueante¹.

- **Yielding**: Es una característica importante en el ESP32 para gestionar tareas de fondo como la conexión Wi-Fi y la pila TCP/IP. Si estas tareas no se atienden, el ESP32 puede reiniciarse. La función `yield()` permite al ESP32 "respirar" y atender estas tareas¹.

- **Designación de Pines**: Debido a la variedad de modelos y placas de desarrollo del ESP32, se recomienda llamar a los pines por su número de GPIO para evitar confusiones¹.

- **E/S Digitales y PWM**: La programación de los GPIO en ESP32 es prácticamente igual que en Arduino. Sin embargo, las salidas analógicas (PWM) funcionan de manera diferente debido a las múltiples formas de generar una señal PWM en el ESP32¹.

- **Entradas Analógicas (ADC)**: Las entradas analógicas se usan de forma similar a Arduino, pero hay diferencias técnicas que deben considerarse¹.

- **Interrupciones**: El ESP32 tiene interrupciones en todos los pines GPIO, lo que ofrece mayor flexibilidad en comparación con Arduino¹.
